shuttle-core

The shuttle-core project consists of various library components that are used to facilitate the development of .Net software.

These components are used extensively in other repositories such as the shuttle-esb and shuttle-recall projects.

NuGet Package Renaming

Within the next couple of weeks all NuGet packages that have the 'shuttle-{package}' structure will be renamed using the .Net assembly naming standard as 'Shuttle.{Package}'.

If you cannot find a particular package using the old structure please remove the old package reference and re-add using the new structure.

Core Components

shuttle-core-host

A generic host that can run as a console application or be installed as a Windows service. Used to host shuttle-esb endpoints but can be used to host any Windows service.

NuGet GitHub Issues

shuttle-core-infrastructure

Contains core infrastructure services such as event-based pipelines, logging, and thread processing.

NuGet GitHub Issues

shuttle-core-data

A very thin abstraction over ADO.NET that enables object-relational mapping.

shuttle-core-infrastructure-log4net
